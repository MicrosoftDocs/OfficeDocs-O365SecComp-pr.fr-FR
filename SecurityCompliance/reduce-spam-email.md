### <a name="for-admins"></a><span data-ttu-id="a30af-115">Pour les administrateurs</span><span class="sxs-lookup"><span data-stu-id="a30af-115">For admins</span></span>

- <span data-ttu-id="a30af-p103">**Pointez vos enregistrements DNS vers Office 365** Pour qu’EOP vous protège efficacement du courrier indésirable, les enregistrements DNS de votre serveur de messagerie (MX) pour tous les domaines doivent pointer vers Office 365 uniquement. - <span data-ttu-id="a30af-p104">**Activez la règle de courrier indésirable dans toutes les boîtes aux lettres** L’action de filtrage du courrier indésirable est définie par défaut sur **Déplacer le message vers le dossier Courrier indésirable**. Pour le vérifier, vous pouvez exécuter la cmdlet Get-MailboxJunkEmailConfiguration dans une ou plusieurs boîtes aux lettres. To check this, you can run the Get-MailboxJunkEmailConfiguration cmdlet against one or more mailboxes. For example, you might check all mailboxes for this by running the following: Get-MailboxJunkEmailConfiguration -Identity \* | Where {$_.Enabled -eq $false}</span></span>
    
    <span data-ttu-id="a30af-p105">Quand le résultat s’affiche, la propriété Enable doit être définie sur True. - <span data-ttu-id="a30af-p106">**Vérifiez vos règles de flux de messagerie et vos listes d’expéditeurs autorisés** Vérifiez l’en-tête d’un message qui aurait dû être marqué comme courrier indésirable. - <span data-ttu-id="a30af-p108">**Marquez le courrier en nombre comme courrier indésirable** Le courrier en nombre désigne le courrier auquel les utilisateurs peuvent être abonnés mais qui est considéré comme indésirable.
