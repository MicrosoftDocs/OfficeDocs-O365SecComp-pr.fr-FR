# <a name="eop-features"></a>Fonctionnalités EOP

Le tableau suivant fournit la liste des fonctionnalités disponibles dans le service de filtrage de courrier électronique hébergé Exchange Online Protection (EOP). 
  
> [!TIP]
> La [feuille de route d'Office 365 pour les entreprises](https://office.microsoft.com/en-us/products/office-365-roadmap-FX104343353.aspx) est une excellente ressource pour trouver des informations sur les nouvelles fonctionnalités à venir. Pour avoir une vision plus large des fonctionnalités disponibles avec les différents plans d'abonnement EOP, voir [Description du service de protection Exchange Online](https://go.microsoft.com/fwlink/p/?LinkId=320619).

|:-----|:-----|
|**Fonctionnalité** <br/> |**Description** <br/> |
|**Protection contre le courrier indésirable** <br/> ||
|Détection de courrier indésirable entrant  <br/> |La protection contre le courrier indésirable entrant est toujours activée et ne peut pas être désactivée. Vous pouvez configurer des paramètres personnalisés via vos stratégies de filtrage des connexions et de contenu.  <br/> Pour les clients autonomes EOP : par défaut, les filtres de contenu EOP envoient des messages de courrier indésirable détectés au dossier courrier indésirable de chaque destinataire. Toutefois, afin de vous assurer que l’action **déplacer le message vers le dossier courrier indésirable** fonctionne avec les boîtes aux lettres locales, vous devez configurer deux règles de transport Exchange sur vos serveurs sur site pour détecter les en-têtes de spam ajoutés par EOP. Pour plus d’informations, voir [vous assurer que le courrier indésirable est routé vers le dossier courrier indésirable de chaque utilisateur](../ensure-that-spam-is-routed-to-each-user-s-junk-email-folder.md).           |
|Détection de courrier indésirable sortant  <br/> |Protection anti-courrier indésirable sortante est toujours activée si vous utilisez le service pour l’envoi de courrier électronique sortant, contribuant ainsi à protéger les organisations qui utilisent le service et leurs destinataires. Comme le filtrage entrant, sortant le filtrage du courrier indésirable est constitué de filtrage des connexions et le filtrage du contenu. Les paramètres de filtrage du courrier indésirable sortant ne sont pas configurables, mais il existe anti-courrier indésirable sortant des paramètres de stratégie que vous pouvez utiliser pour configurer les notifications de l’administrateur pour les messages sortants suspects et bloqués. Pour plus d’informations, voir [configurer la stratégie anti-courrier indésirable sortant](../configure-the-outbound-spam-policy.md).<br/> |
|Protection de rétrodiffusion de rapport de non-remise  <br/> |Pour plus d’informations sur RÉTRODIFFUSION de rapport, voir RÉTRODIFFUSION de définition dans les [options de filtrage avancé du courrier indésirable](../advanced-spam-filtering-asf-options.md) ainsi que des [messages de RÉTRODIFFUSION et EOP](../backscatter-messages-and-eop.md).  <br/> |
|Filtrage de courriers électroniques en masse  <br/> |EOP a amélioré les méthodes de détection de l’identification des messages électroniques en masse. Vous pouvez configurer le service pour marquer les messages électroniques en masse via l’interface utilisateur. Vous pouvez également créer des règles de Transport pour le filtre plus activement les messages en bloc en recherchant un cachet d’en-tête de message de messagerie en bloc. Pour plus d’informations sur les messages électroniques en masse, voir [Quelle est la différence entre courrier indésirable et les messages électroniques en masse ?](../what-s-the-difference-between-junk-email-and-bulk-email.md) et ses sujets secondaires associés.<br/> |
|Listes rouges d'URL malveillantes  <br/> |EOP utilise plusieurs listes rouges d'URL qui permettent de détecter les liens malveillants connus au sein des messages.  <br/> |
|Protection anti-hameçonnage  <br/> |EOP comprend 750 000 domaines d'expéditeurs de courrier indésirable connus.  <br/> |

Pour plus d’informations, voir [configurer vos stratégies de filtrage du courrier indésirable](../configure-your-spam-filter-policies.md).<br/> |
|Possibilité de configurer des actions sur les messages filtrés sur le contenu  <br/> |Il existe plusieurs actions configurables. Par exemple, vous pouvez supprimer des messages de filtre de contenu ou les envoyer vers le dossier courrier indésirable ou la mise en quarantaine. Pour plus d’informations, voir [configurer vos stratégies de filtrage du courrier indésirable](../configure-your-spam-filter-policies.md).<br/> |
|Possibilité de configurer des options avancées de filtrage de courrier indésirable agressif  <br/> |Pour plus d’informations, voir [configurer vos stratégies de filtrage du courrier indésirable](../configure-your-spam-filter-policies.md) (c'est-à-dire vous permet de configurer les) et des [options de filtrage avancé du courrier indésirable](../advanced-spam-filtering-asf-options.md) (qui fournit des détails spécifiques sur chaque option).  <br/> |
|Filtrage de courrier indésirable international  <br/> |Vous pouvez configurer EOP pour filtrer les messages enregistrés dans des langues spécifiques ou envoyés à partir de certains pays ou régions. Pour plus d’informations, voir [configurer vos stratégies de filtrage du courrier indésirable](../configure-your-spam-filter-policies.md).<br/> |
|Gérer le courrier indésirable via Outlook ou Outlook Web App (OWA)  <br/> |Les administrateurs et les utilisateurs finaux peuvent créer des listes d'expéditeurs autorisés et des listes d'expéditeurs bloqués. Pour plus d'informations :  <br/> OWA : voir [Bloquer ou autoriser (paramètres du courrier indésirable)](https://go.microsoft.com/fwlink/p/?LinkId=294862).  <br/> Outlook : voir [À propos du filtre Courrier indésirable](https://go.microsoft.com/fwlink/p/?LinkId=270065).  <br/> Si vous utilisez EOP pour protéger les boîtes aux lettres locales, veillez à utiliser la synchronisation d’annuaires pour vous assurer que ces paramètres sont synchronisées sur le service. Pour plus d’informations sur la configuration de la synchronisation d’annuaires, voir « Utiliser la synchronisation d’annuaires pour gérer les utilisateurs de messagerie » dans [Gérer les utilisateurs de messagerie dans EOP](manage-mail-users-in-eop.md).           |
|Soumission de courriers indésirables via le complément de rapport de courrier indésirable de Microsoft Office Outlook  <br/> |Vous pouvez télécharger un complément pour Outlook qui vous permet d’envoyer des messages de courrier indésirable à Microsoft pour analyse. Pour plus d’informations sur le téléchargement et l’utilisation de cet outil, voir [Activer le complément de Message de rapport](https://support.office.com/article/4250c4bc-6102-420b-9e0a-a95064837676).<br/> Si vous utilisez Exchange Server 2013 avec EOP, vous pouvez également droit dans OWA pour envoyer des messages de courrier indésirable, comme indiqué dans le [courrier indésirable de rapport et de hameçonnage dans Outlook sur le web ](../report-junk-email-and-phishing-scams-in-outlook-on-the-web-eop.md).  <br/> |
|Soumission de courriers indésirables et de courriers légitimes via un alias de messagerie  <br/> |Vous pouvez envoyer du courrier indésirable (indésirable) et non (légitime) de courrier indésirable à Microsoft par courrier électronique. Pour plus d’informations, voir [envoi spam, légitimes et des hameçonnage anti-spam à Microsoft pour analyse](../submit-spam-non-spam-and-phishing-scam-messages-to-microsoft-for-analysis.md).<br/> |
|Soumission de courriers indésirables et de courriers légitimes via le signalement de courrier indésirable avec OWA  <br/> |Vous pouvez envoyer du courrier indésirable et non de courrier indésirable à Microsoft via OWA Junk Email Reporting. Pour plus d’informations, voir [courrier indésirable de rapport et de hameçonnage dans Outlook sur le web ](../report-junk-email-and-phishing-scams-in-outlook-on-the-web-eop.md).<br/> Cette fonctionnalité est actuellement disponible pour les clients Outlook Web App (OWA) dont les boîtes aux lettres Exchange Server 2013 SP1 sont filtrées par EOP. Les clients OWA Exchange Online disposeront eux-aussi bientôt de cette fonctionnalité.           |
|Notifications de mise en quarantaine du courrier indésirable pour les utilisateurs finaux  <br/> |Les utilisateurs finaux peuvent libérer leurs propres messages mis en quarantaine du courrier indésirable et les signaler comme légitime par le biais de messages de notification de courrier indésirable de l’utilisateur final. Pour plus d'informations, voir [Find and release quarantined messages as an administrator](../find-and-release-quarantined-messages-as-an-administrator.md) ou [Find and Release Quarantined Messages as an End User](http://technet.microsoft.com/library/e439b560-827a-4807-abd3-6b861c1ff786.aspx).  <br/> |
|Accéder aux messages en quarantaine et les gérer via une page web  <br/> |Disponible pour les utilisateurs finaux et les administrateurs. Pour plus d'informations, voir [Find and release quarantined messages as an administrator](../find-and-release-quarantined-messages-as-an-administrator.md) ou [Find and Release Quarantined Messages as an End User](http://technet.microsoft.com/library/e439b560-827a-4807-abd3-6b861c1ff786.aspx).  <br/> |

|**Protection anti-programme malveillant** <br/> ||
|Protection anti-programme malveillant multi-moteur  <br/> |Plusieurs moteurs anti-programme malveillant aident à protéger automatiquement nos clients à tout moment.  <br/> |

Par conséquent, le filtrage des programmes malveillants est automatiquement activé pour tous les clients.  <br/> |

Ces notifications sont envoyées uniquement lorsque l’intégralité du message est supprimé. Pour plus d’informations, voir [configurer les stratégies anti-programme malveillant](../configure-anti-malware-policies.md).<br/> |
|Possibilité de supprimer une pièce jointe en cas de détection de programme malveillant  <br/> |Les administrateurs peuvent sélectionner s’il faut supprimer l’intégralité du message ou de détacher la pièce jointe et envoyer un message personnalisé pour les destinataires. Stratégies personnalisées sont toujours prioritaires sur la stratégie par défaut, mais vous pouvez modifier la priorité (autrement dit, l’ordre en cours d’exécution) de vos stratégies personnalisées. |**Règles de transport** <br/> ||
|Filtrage et actions basés sur une stratégie  <br/> |Stratégies personnalisées sont basées sur des règles de Transport Exchange. Vous pouvez filtrer par domaine, mot clé, nom de fichier, type de fichier, ligne d’objet, le corps du message, expéditeur, destinataire, en-tête et adresse IP. Pour plus d’informations, consultez la rubrique [Mail flow (règles de transport) dans Exchange Online Protection](mail-flow-rules-transport-rules-0.md).<br/> |
|Filtrage par modèles de texte  <br/> |Les règles de transport peuvent utiliser un tableau ou des expressions régulières pour établir une correspondance avec le texte. Vous pouvez également utiliser une chaîne ou un tableau de chaînes pour établir une correspondance avec de nombreuses propriétés de message, telles que l’adresse, l’objet, le corps ou les noms de pièce jointe. Pour plus d’informations, voir [Transport Rule Conditions (Predicates)](http://technet.microsoft.com/library/04edeaba-afd4-4207-b2cb-51bcc44e483c.aspx).<br/> |
|Dictionnaires personnalisés  <br/> |Les règles de transport Exchange peuvent contenir de longues listes de texte et de mots clés, offrant la même fonctionnalité qu'un dictionnaire personnalisé.  <br/> |
|Règles de stratégie par domaine  <br/> |La portée d’une règle de transport peut être personnalisée pour correspondre aux noms de domaine de l’expéditeur ou du destinataire, aux plages d’adresses IP, aux mots clés ou aux modèles d’adresse, à l’appartenance à des groupes et à d’autres conditions. Pour plus d’informations, voir [Transport Rule Conditions (Predicates)](http://technet.microsoft.com/library/04edeaba-afd4-4207-b2cb-51bcc44e483c.aspx).<br/> |
|Analyse de pièce jointe  <br/> |Des règles peuvent être créées pour analyser le nom du fichier, son extension et le contenu de la pièce jointe.  <br/> |
|Envoyer les notifications de règle de stratégie à l'expéditeur  <br/> |Vous pouvez rejeter des messages et envoyer un rapport de non-remise à l’expéditeur via l’action **Rejeter le message avec l’explication** ou **Rejeter le message avec le code d’état amélioré**. Pour plus d’informations, voir [Transport rule actions](http://technet.microsoft.com/library/f8621ecb-a177-4025-9011-a6569999746a.aspx).<br/> |
|Envoyer des messages à des adresses fixes (par exemple, redirection ou copie d'un message vers une adresse spécifique)  <br/> |Les règles de transport peuvent rediriger, ajouter des destinataires en copie carbone ou en copie carbone invisible, ajouter des destinataires de façon simple, etc. Pour plus d’informations, voir [Transport rule actions](http://technet.microsoft.com/library/f8621ecb-a177-4025-9011-a6569999746a.aspx).<br/> |
|Possibilité d'ajuster facilement la priorité des règles sur plusieurs règles  <br/> |Utilisez le Centre d'administration Exchange pour modifier l'ordre de traitement des règles. Pour plus d'informations, voir [Manage Transport Rules](http://technet.microsoft.com/library/e7a81372-b6d7-4d1f-bc9e-a845a7facac2.aspx).  <br/> |
|Possibilité de filtrer les messages, puis de modifier le routage ou les attributs d'un message  <br/> |Vous pouvez filtrer les messages selon un large éventail de conditions, puis appliquer une série d’actions à chaque message. Pour plus d’informations, consultez la rubrique [Mail flow (règles de transport) dans Exchange Online Protection](mail-flow-rules-transport-rules-0.md).<br/> |
|Modifier le seuil de probabilité de courrier indésirable d'un message par règle  <br/> |Vous pouvez inspecter un message en transit et affecter un niveau de confiance du courrier indésirable en fonction de critères que vous choisissez. Pour plus d’informations, voir [utiliser les règles de flux de messagerie pour définir le niveau de confiance du courrier indésirable (SCL) dans les messages](../use-mail-flow-rules-to-set-the-spam-confidence-level-scl-in-messages.md).<br/> |
|Inspecter les pièces jointes à un message  <br/> |Vous pouvez examiner le contenu d'une pièce jointe ou les caractéristiques d'un fichier joint, puis définir l'action à effectuer en fonction de ce que vous avez trouvé. Pour plus d'informations, voir [Using transport rules to inspect message attachments](http://technet.microsoft.com/library/874d1c78-a8ec-4938-b388-d3208c2fa971.aspx).  <br/> |

|**Création de rapports et journalisation** <br/> ||
|Suivi des messages  <br/> |La fonctionnalité de suivi des messages vous permet, en tant qu'administrateur, de suivre des messages électroniques lorsqu'ils sont acheminés via le service. Vous pouvez ainsi répondre efficacement aux questions de vos utilisateurs, résoudre les problèmes de flux de messagerie et valider les modifications de stratégie, tout en réduisant la nécessité de demander de l'aide à l'assistance technique. Avec ces rapports interactifs, vous pouvez afficher rapidement un rapport des données de synthèse et accéder à des informations plus détaillées sur des messages individuels remontant jusqu'à 90 jours. Pour plus d'informations, voir [Use mail protection reports in Office 365 to view data about malware, spam, and rule detections](http://technet.microsoft.com/library/bcef7984-4bfa-4ca8-9fa5-a65af8618f5d.aspx).  <br/> |
